## Meeting Lingo — Phrases That Sound Natural

### Instead of textbook language, say:

| ❌ Textbook                                   | ✅ What engineers actually say                                    |
| --------------------------------------------- | ----------------------------------------------------------------- |
| "The BGP session is in Established state"     | "BGP is up" or "the session is up"                                |
| "The BGP session is not in Established state" | "BGP is down" or "the session is flapping"                        |
| "The OSPF neighbor relationship is FULL"      | "OSPF is up" or "we're FULL with both spines"                     |
| "I will examine the routing table"            | "Let me check the routes" or "let me look at the RIB"             |
| "The route is being advertised via BGP"       | "We're advertising that prefix" or "that route is being exported" |
| "Configure BFD on the interface"              | "Enable BFD" or "turn on BFD"                                     |
| "The autonomous system number is 65000"       | "We're in AS sixty-five thousand"                                 |
| "We need to examine the EVPN route types"     | "Let me check the EVPN table"                                     |
| "The MTU is misconfigured"                    | "There's an MTU mismatch"                                         |
| "I will generate the configuration"           | "I'll build the config" or "I'll put the config together"         |
| "We should implement route summarization"     | "We should summarize at the border"                               |
| "The forwarding table"                        | "The FIB" (rhymes with "rib")                                     |
| "The routing information base"                | "The RIB" (rhymes with "bib")                                     |
| "I need to check the link-state database"     | "Let me look at the LSDB"                                         |
| "The Network Layer Reachability Information"  | Just say "the NLRI" or "the prefix"                               |
| "Virtual Routing and Forwarding instance"     | Just say "VRF" — nobody says the full name                        |

### Casual shorthand engineers use:

| Shorthand                           | Meaning                                                         |
| ----------------------------------- | --------------------------------------------------------------- |
| "The session is bouncing"           | BGP/OSPF flapping up and down                                   |
| "We're leaking routes"              | Routes appearing in a VRF they shouldn't be in                  |
| "That prefix is getting blackholed" | Traffic is being sent somewhere it gets dropped                 |
| "We've got a loop"                  | Routing loop — traffic is going in circles                      |
| "The fabric is converging"          | Routing tables are updating after a change                      |
| "Hit the pre-checks"                | Run pre-change state capture                                    |
| "Diff the state"                    | Compare pre and post change captures                            |
| "Push the config"                   | Apply configuration to devices                                  |
| "Roll it back"                      | Revert to previous config                                       |
| "Clear the session"                 | `clear bgp neighbor X.X.X.X` — reset BGP                        |
| "Bounce the interface"              | Shut / no shut                                                  |
| "We're peered with Dallas"          | We have a BGP session to the Dallas site                        |
| "That's in the CDE VRF"             | That traffic/route lives in the PCI cardholder data environment |
| "The spines are RRs"                | The spine switches are BGP route reflectors                     |
